# 分块评估功能 — 设计与交互

> 基于 `ui-ux-pro-max` 设计系统 (MASTER.md) 的 KB 管理页面扩展。
> 风格继承：IBM Plex Sans / 蓝色主色 #2563EB / 财务专业调性

---

## 1. 功能概述

在文档列表（KB 管理页）中，对有评估数据的文件展示 **分块质量评分**，并提供 **两级弹窗** 查看详情：

- **文档行**：显示 `分块评分 0.89 ✓/✗`，绿色通过/红色不通过
- **一级弹窗（评估总览）**：综合评分 + 三个模块的分数和断裂链接
- **二级弹窗（断裂详情）**：具体断裂内容的预览

## 2. 文档列表行 — UI 改动

### 当前布局（原始代码）

```
┌──────────────────────────────────────────────────────────────┐
│  PDF   filename.pdf    1.2MB · 14 分块 · 2026/7/12          │
│                                    ✓ 就绪       [🗑]         │
└──────────────────────────────────────────────────────────────┘
```

### 改动后布局

```
┌──────────────────────────────────────────────────────────────┐
│  PDF   filename.pdf    1.2MB · 14 分块 · 2026/7/12          │
│                           分块评分 0.89 ✓                    │
│                                    ✓ 就绪  [详情]  [🗑]      │
└──────────────────────────────────────────────────────────────┘
```

### 视觉规格

| 元素 | 样式 | 说明 |
|------|------|------|
| 评分文字 | `text-xs font-bold` | 紧跟在文件信息下方 |
| 通过颜色 | `text-emerald-600` | score ≥ 0.70 时绿色 |
| 未通过颜色 | `text-red-500` | score < 0.70 时红色 |
| 详情按钮 | `px-2 py-1 text-xs text-blue-600 hover:bg-blue-50 rounded-md` | 有评估数据时才显示 |
| 行背景（未通过） | `bg-red-50/30` 行级别浅红背景 | 视觉提示该文件需要关注 |

### 交互

- 鼠标悬停详情按钮 → 轻微背景变化 (`hover:bg-blue-50`)
- 点击详情 → 打开一级弹窗（`eval-modal`）
- 单击文档行（非详情/删除区域）→ 打开分块预览（保持原行为）

| 模块标题 | `text-xs font-semibold text-slate-500 uppercase tracking-wider` | 小号大写 |
| 进度条背景 | `h-1.5 bg-slate-100 rounded-full` | 浅灰轨道 |

| 断裂链接 | `text-xs text-blue-600 hover:text-blue-700 font-medium underline underline-offset-2` | "N个断裂 → 查看" |
| 无断裂文字 | `text-xs text-slate-400` | "无断裂" |

### 分数颜色阈值

| 区间 | 颜色 | 说明 |
|------|------|------|
| ≥ 0.70 | `text-emerald-600` / `bg-emerald-500` | 绿色 - 通过 |
| 0.50 ~ 0.69 | `text-amber-600` / `bg-amber-500` | 黄色 - 需关注 |
| < 0.50 | `text-red-500` / `bg-red-500` | 红色 - 不通过 |

### 交互

- 点击弹窗外区域 / 点击 ✕ 按钮 / 按 ESC → 关闭弹窗
- 点击「N个断裂 → 查看」→ 打开二级弹窗
- 点击「无断裂」→ 无交互

## 6. 上传中弹窗

### 原始问题

Tailwind CDN 异步加载导致 `hidden` 类闪动，弹窗在页面渲染时短暂可见。

### 修复方案

所有弹窗使用 `style="display:none"` 替代 `class="hidden"`，JS 用 `style.display = 'block'` / `'none'` 控制显隐。

```html
<!-- 修复前 -->
<div id="upload-loading" class="fixed inset-0 z-50 hidden">

<!-- 修复后 -->
<div id="upload-loading" class="fixed inset-0 z-50" style="display:none">
```

此改动应用于 upload-loading、eval-modal、eval-broken-modal 三个弹窗。

## 7. 响应式考虑

| 断点 | 弹窗宽度 | 文档行布局 |
|------|---------|-----------|
| < 640px | 全屏（`inset-0`） | 垂直堆叠，评分和按钮在文件名下方 |
| 640-1024px | `w-[90vw]` | 横向排列，评分在文件信息行内 |
| > 1024px | `max-w-2xl` | 横向排列 |

## 8. 设计决策记录

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 评分展示位置 | 文件名下方 / 状态标签旁 | 文件名下方 | 不干扰已有的状态标签和操作按钮 |
| 弹窗还是tooltip | tooltip / 弹窗 | 弹窗 | 评估数据较多（综合分+3模块+断裂明细），tooltip 无法承载 |
| 一级/二级弹窗拆分 | 单弹窗展开 / 两级 | 两级 | 二级弹窗专看断裂明细，保持一级弹窗清晰聚焦综合评分 |
| 进度条 vs 纯数字 | 进度条+数字 / 纯数字 | 进度条+数字 | 进度条视觉直观，数字精确，两者互补 |
